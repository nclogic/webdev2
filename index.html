HTML Structure

HTML, which stands for HyperText Markup Language, is the foundation of web pages. It uses various tags to structure and organize content.This includes the <html>, <head>, and <body> tags. The <head> tag typically contains meta information about the document, such as its title and link to CSS files. The <body> tag contains the content of the document.

Understanding the basics of HTML structure is crucial for web design.

Links and Anchors

Links and anchors are integral elements of HTML, enabling users to navigate and interact with web content. Here, we explore their role and usage.

Hyperlinks (<a> Tags)

Hyperlinks, often referred to as links, are created using the <a> (anchor) element. They allow you to connect web pages and resources, enabling users to move seamlessly between different parts of your website.

Anchor Tags with id

Anchor tags can also be enhanced with an id attribute. This creates anchor points within a page, making it possible to link to specific sections of the same page. For example, you can create a table of contents with links to different sections on a long webpage.

External and Internal Links

When creating links, you can use absolute URLs for external resources, such as https://www.example.com/page. For internal links within your website, relative URLs are often used. Relative URLs specify the path from the current page to the target page.

External Link Example: <a href="https://www.google.com">Google</a> This would take a user to Google when they click the link.

Internal Link Example: <a href="#section-two">Section 2</a> This will take a user to an internal part of the website identified by it's id value.

Target Attribute

The target attribute in the anchor tag controls how links open. For instance, using target="_blank" opens the linked content in a new browser tab or window.

<a href="https://www.google.com" target="_blank">Google</a>

Understanding the functionality of links and anchors is essential for effective website navigation and user experience. They play a crucial role in guiding visitors through your content.

Lists (Ordered and Unordered)

Ordered Lists (<ol>)

Ordered lists are used to present content in a numbered sequence. Each list item is automatically assigned a number or another ordered marker.

Example:

  1. <ol>
  2.   <li>Get fuel</li>
  3.   <li>Stop by the Computer Store</li>
  4.   <li>Stop by the grocery store</li>
  5. </ol>

This displays on the web page like:

  1. Get fuel
  2. Stop by the Computer Store
  3. Stop by the grocery store

Unordered Lists (<ul>)

Unordered lists are used to present content with bulleted or other unordered markers. They are often used for lists without a specific order or sequence.

Example:

  • <ul>
  •   <li>Stop by the Computer Store</li>
  •   <li>Get fuel</li>
  •   <li>Stop by the grocery store</li>
  • </ul>

This displays on the web page like:

  • Stop by the Computer Store
  • Get fuel
  • Stop by the grocery store

Like this: <img src="/images/cats.jpg" alt="Image of cats"> Remember; <img> tags are self closing, meaning they do not require a matching </img>

Tables (<table>)

Tables are used to organize data in rows and columns. The <table> element is the container for creating tables. You can add rows with <tr> and define headers with <th>.

Example:

  •     <th>Header 2</th>
  •   </tr>
  •   <tr>
  •     <td>Data 1</td>
  •     <td>Data 2</td>
  •   </tr>
  • </table>

It displays on the page like this:

    • Doctype Declaration

    The Doctype Declaration is the very first line in an HTML document. Its primary purpose is to inform the web browser about the type of document format it will be rendering.

    <!DOCTYPE html>  This declaration is a mandatory and essential component of an HTML document, always placed at the document's beginning.

    It is immediately followed by the opening <html lang="en">  tag, which includes the lang attribute specifying the language of the document. For example, "en" is used for English, while "fr" represents French, and "es" stands for Spanish.

    The tag, along with its lang attribute, sets the language for the entire document and ensures proper rendering and language support. This line always appears as the second line in the HTML structure.

    Viewport Meta Tag

    The viewport meta tag plays a crucial role in achieving responsive web design.

    It provides essential instructions to the web browser on how to manage the page's dimensions and scaling, ensuring that web content adapts effectively to various screen sizes and devices.

    <meta name="viewport" content="width=device-width, initial-scale=1.0" /> (notice this is a self closing element tag)

    CSS Reset

    A CSS reset is a collection of CSS rules designed to reset the default styling of HTML elements to a consistent baseline. It serves to eliminate any discrepancies or variations in rendering across different web browsers, ensuring a more uniform and predictable appearance of web content.

    Layout (grid System)

    A grid system is an invaluable tool for achieving balanced layout designs. It establishes uniform margins and gutters, providing a structured framework for organizing and placing each piece of content. This systematic approach simplifies the design process and enhances the overall visual consistency of your web pages.

    Typogrophy

    Establish the foundation of your typography by configuring the base font size, line-height, and font family. These elements serve as the fundamental building blocks of your text styles, and they can be customized to align with your specific design requirements.

    Colors

    Define your foundational color scheme, encompassing primary, secondary, and accent colors that will underpin the visual identity of your website.

    Headers & Footers

    Header: Typically, a website's header features key elements such as the site logo, navigation menu, and sometimes a search bar or a prominent call-to-action button. As one of the first elements visitors encounter, it plays a crucial role in delivering information and ensuring seamless navigation.

    Footer: In contrast, the footer generally houses essential information like contact details, links to social media profiles, and often a sitemap. It serves as the final point of interaction, providing valuable supplementary content and connections.

    Commenting Code

    Adding comments to your code is Important. It helps in several ways:

    • Makes code human-readable and elegant: Comments can explain what the code does and clarify the decisions behind code design. They make the code easier to understand for yourself and others.
    • Facilitates debugging: Comments can be used to hide your HTML, CSS, and JavaScript code, or to temporarily disable code while testing.
    • Helps in team collaboration: They allow your team to understand and edit your code easily.
    • Documents your thought process: Comments help you document and communicate about your code and thought process to yourself (and others).
    • Aids future understanding: They remind you what you were thinking/doing when you come back to a project after months of not working on it.
